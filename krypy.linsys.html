krypy.linsys - Linear Algebraic Systems Solver¶

The linsys module provides functions for the solution of linear algebraic systems.

krypy.linsys.cg(A, b, x0=None, tol=1e-05, maxiter=None, M=None, Ml=None, Mr=None, inner_product=<function ip_euclid at 0x2aaab596f560>, explicit_residual=False, return_basis=False, full_reortho=False, exact_solution=None)¶

Conjugate gradient method with different inner product.

krypy.linsys.gmres(A, b, x0=None, tol=1e-05, maxiter=None, M=None, Ml=None, Mr=None, inner_product=<function ip_euclid at 0x2aaab596f560>, explicit_residual=False, return_basis=False, full_reortho=True, exact_solution=None, max_restarts=0)¶

Preconditioned GMRES

Solves \(M M_l A M_r y = M M_l b\) where \(x=M_r y\).

Memory consumption is about maxiter+1 vectors for the Arnoldi basis. If \(M\) is used the memory consumption is 2*(maxiter+1).

Parameters:
  • A – a linear operator on \(\mathbb{C}^n\).
  • b – a vector in \(\mathbb{C}^n\).
  • tol –

    the tolerance for the stopping criterion with respect to the relative residual norm:

    \[\frac{ \| M M_l (b-A (x_0+M_r y_k))\|_{M^{-1}} }{ \|M M_l b\|_{M^{-1}}} \leq \text{tol}\]
  • maxiter – maximum number of iterations per restart cycle, see max_restarts. Has to fulfill \(\text{maxiter}\leq n\).
  • M – a self-adjoint and positive definite linear operator on \(\mathbb{C}^n\) with respect to inner_product. The changes the inner product used for orthogonalization to \(\langle x,y\rangle_M = \langle Mx,y\rangle\) where \(\langle \cdot,\cdot\rangle\) is the inner product defined by the parameter inner_product. If set to None then \(M\) is the identity.
  • Ml – left preconditioner, linear operator on \(\mathbb{C}^n\). If set to None then \(M_l\) is the identity.
  • Mr – right preconditioner, linear operator on \(\mathbb{C}^n\). If set to None then \(M_r\) is the identity.
  • inner_product – a function that takes two arguments and computes the (block-) inner product of the arguments.
  • explicit_residual – if set to False, the updated residual norm from the GMRES iteration is used in each iteration. If set to True, the residual is computed explicitly in each iteration and thus requires an additional matrix-vector multiplication in each iteration.
  • return_basis – if set to True then the computed Arnoldi basis and the Hessenberg matrix are returned in the result dictionary with the keys V and H. In exact
  • exact_solution – if the solution vector \(x\) is passed then the error norm \(\|x-x_k\|\) will be computed in each iteration (with respect to inner_product) and returned as a list in the result dictionary with the key errvec.
  • max_restarts – the maximum number of restarts. The maximum number of iterations is (max_restarts+1)*maxiter.
Returns:

a dictionary with the following keys:

  • xk: the approximate solution \(x_k\).
  • info: convergence flag (0 if converged, 1 otherwise).
  • relresvec: relative residual norms of all iterations, see parameter tol.
  • V: present if return_basis=True. The Arnoldi basis vectors.
  • H: present if return_basis=True. The Hessenberg matrix.
  • P: present if return_basis=True and M is provided. The matrix \(P\) fulfills \(V=MP\).
krypy.linsys.minres(A, b, x0=None, tol=1e-05, maxiter=None, M=None, Ml=None, Mr=None, inner_product=<function ip_euclid at 0x2aaab596f560>, explicit_residual=False, return_basis=False, full_reortho=False, exact_solution=None, timer=False)¶

Preconditioned MINRES

This MINRES solves M*Ml*A*Mr*y = M*Ml*b, x=Mr*y where Ml and Mr have to be such that Ml*A*Mr is self-adjoint in the inner_product. M has to be self-adjoint and positive-definite w.r.t. inner_product.

Details: The Lanczos procedure is used with the operator M*Ml*A*Mr and the inner product defined by inner_product(M^{-1}x,y). The initial vector for Lanczos is r0 = M*Ml*(b - A*x0) – note that Mr is not used for the initial vector!

Stopping criterion is ||M*Ml*(b-A*(x0+Mr*yk))||_{M^{-1}} / ||M*Ml*b||_{M^{-1}} <= tol
